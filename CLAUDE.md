# VBGone Output — Claude Code Context

## What This Repo Is

This is the output repository for VBGone. It contains C# code generated by the VBGone app — interfaces, NUnit test suites, and implementations migrated from legacy VB.NET. Each migration lives in its own directory under `generated/`.

The companion app that generates and commits code to this repo lives at `../vbgone-app/`.

## Structure

Each migrated class follows the same layout:

```
generated/<ClassName>/
├── <ClassName>.sln
├── <ClassName>/
│   ├── <ClassName>.csproj
│   ├── I<ClassName>.cs          ← interface (contract)
│   └── <ClassName>.cs           ← implementation
└── <ClassName>.Tests/
    ├── <ClassName>.Tests.csproj
    └── <ClassName>Tests.cs      ← NUnit test suite
```

## Tech Stack

- .NET 10.0 (net10.0)
- NUnit 4 + NUnit3TestAdapter
- Coverlet for code coverage
- Stryker.NET for mutation testing (nightly)
- CodeQL for security scanning

## Conventions

- One solution per migrated class
- Interface name: `I<ClassName>`
- Test class name: `<ClassName>Tests`
- Test project references the main project
- All projects target `net10.0`
- NUnit is the test framework — see `docs/decisions/001-nunit-over-xunit.md`
- Warnings treated as errors in Release configuration

## CI Pipeline

- `.github/workflows/ci.yml` — runs on every PR: build, test, coverage, CodeQL, CVE scan
- `.github/workflows/mutation.yml` — Stryker.NET runs nightly against main

## Commands

```bash
# Build and test a specific migration
cd generated/ArithmeticOperations
dotnet build
dotnet test

# Run with coverage
dotnet test --collect:"XPlat Code Coverage" --logger "trx;LogFileName=results.trx"

# Run Stryker locally
cd generated/ArithmeticOperations
dotnet stryker
```

## Important Notes

- VBGone raises PRs against this repo programmatically — do not change the directory structure convention
- The GitHub Actions workflows expect solutions under `generated/*/`
- Do not add project-level Stryker config — it is configured in the workflow
